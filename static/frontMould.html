固证浏览器

1. 开发环境

  • VS2013+
  • .NET Framework,Version=v4.5
  • Windows 6+

2. 项目开始

  • 安装VS2017 地址
  • Git 项目地址

3. 项目结构

  • GZ.Browser 固证浏览器核心业务模块，包含用户登录、录屏、截屏、网络数据校验、固证、固证管理，浏览器使用IE Trident内核。
  • GZ.AutoUpdater 固证浏览器自动更新模块，包含版本检查、下载、更新。
  • GZ.Browser.Dao 固证浏览器DAO模块，包含本地数据库（SQLite）实体对象、NHibernate Mapping文件。
  • GZ.UI 固证浏览器页面基础模块。

4. 项目开发

  • Step1 右键 \<GZ.Browser> 设定为“起始项目”。
  • Step2 展开 \<GZ.Browser> 于Properties设定签名信息。
  • 程序入口文件Program.cs。
  • 其余开发按照各模块功能划分进行开发。

5. 业务流程

  • 1、首先通过WebBrowser 组件加载一个网页，同时调用DnsPackage 去解析网址的域名，获得域名的安全等级（分别为：安全、警告、危险）

    安全：通过制定DNS服务解析域名的IP和本地解析一致

    警告：通过制定DNS服务解析的域名和本地解析不一致

    危险：本地有DNS代理或Host文件中有配置该域名指向

  • 2、点击截取按钮触发截屏事件-该事件可订阅（分为单和图片截取和多个图片截取）。

  • 3、截图消费者收到截图消息并接收ActionInfo 对象实体，ActionInfo 包括 UserId、ActionId、截图句柄、和DNS安全信息，保存文件，并向服务器请求一个evidenceId，成功则产生一个Evidence实例并通知打包消费者PackObserver。

  • 4、签名SignObserver消费者，同样收到Evidence消息获得打包文件路径，生成PDF文件并签名。

  • 5、打包 PackObserver消费者收到Evidence消息获得图片保存路径使用打包工具，产生7z文件。

  • 6、后台通过定时任务UploadFileTimer_Tick 上传已7z文件.

6. 项目打包

  • 打包工具 地址
  • 打包 脚本
  • 打包流程
    • 拷贝 <项目路径>\GZ.Browser\GZ.Browser\bin\Debug 至 \ 。
    • 拷贝 <项目路径>\GZ.Browser\GZ.Browser\Tools 至 \ \Tools。
    • 移除\ 中pdb文件。
    • 修改打包脚本OutputDir字段，修改【Files】中的Source为上一步的\ 。

7. 项目发布

  • 更新GZ.Browser\Properties\Settings.settings的VersionManagerServer为线上字段。
  • 更新GZ.Browser\Properties\AssemblyInfo.cs的AssemblyVersion为当前版本。
  • 更新打包脚本AppId、AppVersion。
  • 将EXE安装文件及\ 的zip压缩文件推送到百度云。
